# IC-Logic-Tester
Tester for logic chips

Тестер для микросхем логики на PIC18F452. Создавался с целью попробовать свои 
силы на этом поприще, а так же утилизировать ненужные компоненты, поскольку к 
тому времени я уже взял курс на AVR и ARM. Отсюда и такой странный выбор 
микроконтроллера, не очень подходящего для подобного устройства, пришлось даже 
рэле для управления питанием задействовать. 
Изначально набор тестируемых микросхем был сильно ограничен, но уже после 
создания платы нашел хороший тестер микросхем, с открытым исходным кодом (сейчас 
частично открыт): 
https://trolsoft.ru/ru/sch/chip-tester
откуда я позаимствовал таблицы тестов (набор кодов, с помощью которых программа 
знает на какие ножки что подать и с каких что считать) для остальных микросхем 
логики, за что автору огромное спасибо. Правда пришлось их сильно переделать,
в виду кардинального различия микроконтроллеров и схем приборов. Всё остальное 
моё, включая процесс отображения и логику тестирования. Плюс добавил тесты 
динамической памяти. На большее просто не хватает памяти. 
Все вспомогательные утилиты находятся в папке TOOLS, а их описание можно
найти там же, в файле README.TXT.

Процесс тестирования отображается графически: направление ножек (ввод/вывод), 
уровень сигнала ("1" (красный) или "0" (синий)). При ошибке ножки, на которых
зафиксирован неверный результат, выделяются прямоугольником и на экран выводится
код ошибки, а точнее порядковый номер теста.
Имеется так же пошаговый режим тестирования. Для этого нужно нажать клавишу
Enter и включить прибор. На экране отобразится надпись "DEBUG MODE: ON".
В этом режиме после каждой команды теста необходимо нажать Enter для продолжения.
Тест не прерывается даже в случае обнаружения ошибок, а будет пройден до конца.

Поскольку прибор максимально упрощен, то некоторым микросхемам не хватает
тока. В частности, серия 589 вообще не тянет. А для серии 580 приходится
дополнительно цеплять землю. Список протестированных микросхем в файле 
README.TXT. Там же и примечания по проблемным типам.

Исходники компилировать только HI-TECH C Pro for PIC18. Только версия PRO.
Не знаю почему, но версия STD создаёт нерабочий код. Возможно нужно убрать
настройки оптимизации.
